**Telnet** — протокол для удалённого доступа к командной строке сервера. Telnet передаёт все данные (включая пароли) в открытом виде, поэтому небезопасен. В современных системах заменён на SSH.

## **Основные характеристики**

| Параметр                    | Описание                                                                 |
| --------------------------- | ------------------------------------------------------------------------ |
| **Уровень OSI**             | Прикладной (Application Layer)                                           |
| **Транспорт**               | TCP (порт 23)                                                             |
| **Тип протокола**           | С установлением соединения                                              |
| **Безопасность**            | Небезопасен (данные в открытом виде)                                     |
| **Статус**                  | Устаревший, заменён на SSH                                               |

## **Структура Telnet**

### **Команды и данные**

Telnet передаёт команды и данные в виде байтов:
- **Данные** — обычные символы
- **Команды** — специальные последовательности (IAC - Interpret As Command)

### **IAC команды**

| Код | Команда          | Описание                                    |
| --- | ---------------- | ------------------------------------------- |
| 240 | IAC              | Interpret As Command                        |
| 241 | NOP              | No Operation                                |
| 242 | Data Mark        | Data Mark                                   |
| 243 | Break            | Break                                         |
| 244 | Interrupt Process | Прервать процесс                          |
| 245 | Abort Output     | Прервать вывод                              |
| 246 | Are You There    | Проверка наличия                            |
| 247 | Erase Character  | Стереть символ                              |
| 248 | Erase Line       | Стереть строку                              |
| 249 | Go Ahead         | Продолжить                                  |
| 250 | SB               | Subnegotiation Begin                        |
| 251 | WILL             | Согласие на опцию                            |
| 252 | WONT             | Отказ от опции                              |
| 253 | DO               | Запрос на опцию                              |
| 254 | DONT             | Отказ от опции                              |

## **Примеры использования**

### **Командная строка**

```bash
# Подключение
telnet server.example.com 23

# С указанием порта
telnet server.example.com 80

# Проверка доступности порта
telnet server.example.com 22
```

### **Интерактивная сессия**

```
$ telnet server.example.com
Trying 192.168.1.100...
Connected to server.example.com.
Escape character is '^]'.

Linux server 5.4.0
login: username
Password:
$
```

## **Безопасность Telnet**

### **Проблемы**

1. **Данные в открытом виде** — пароли передаются незашифрованными
2. **Уязвим к перехвату** — легко перехватить трафик
3. **Нет аутентификации соединения** — уязвим к MITM атакам

### **Рекомендации**

**НЕ использовать Telnet для:**
- Удалённого управления
- Передачи паролей
- Любых чувствительных данных

**Использовать вместо:**
- **SSH** — безопасный протокол
- **HTTPS** — для веб-доступа
- **VPN** — для защищённого доступа

## **Применение Telnet**

1. **Отладка** — проверка доступности портов
2. **Тестирование** — тестирование текстовых протоколов
3. **Legacy системы** — старые системы, где SSH недоступен
4. **Локальные сети** — в изолированных сетях

## **Диагностика с Telnet**

### **Проверка портов**

```bash
# Проверка HTTP
telnet server.example.com 80
GET / HTTP/1.1
Host: server.example.com

# Проверка SMTP
telnet mail.example.com 25
EHLO example.com

# Проверка POP3
telnet mail.example.com 110
USER username
PASS password
```

## **Ограничения Telnet**

1. **Небезопасность** — главная проблема
2. **Устаревание** — заменён на SSH
3. **Ограниченная функциональность** — базовые возможности

## **Миграция на SSH**

### **Замена Telnet на SSH**

```bash
# Вместо telnet использовать ssh
ssh user@server.example.com

# Автоматизация
ssh user@server.example.com "command"
```

## **Будущее Telnet**

- **Устаревание** — постепенное прекращение использования
- **Legacy поддержка** — только для старых систем
- **Замена на SSH** — во всех новых системах
