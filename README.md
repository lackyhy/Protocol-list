HTTP:

Коды из диапазона 1xx классифицируются как информационные.
(1xx: Informational)

| **Код** | **Статус**          | **Описание**                                      |
| ------- | ------------------- | ------------------------------------------------- |
| 100     | Continue            | Запрос принят, клиент должен продолжать отправку. |
| 101     | Switching Protocols | Сервер переключается на другой протокол.          |
| 102     | Processing          | Запрос в обработке.                               |
| 103     | Early Hints         | Ранние подсказки, предзагрузка ресурсов.          |

2xx коды указывают на успешное выполнение запросов.
(2xx: Success)

| Код | Статус                        | Описание                                                                                                      |
| --- | ----------------------------- | ------------------------------------------------------------------------------------------------------------- |
| 200 | OK                            | Запрос успешно обработан.                                                                                     |
| 201 | Created                       | Ресурс успешно создан.                                                                                        |
| 202 | Accepted                      | Запрос принят, но обработка не завершена.                                                                     |
| 204 | No Content                    | Запрос выполнен, контент отсутствует.                                                                         |
| 206 | Partial Content               | Частичное содержимое.                                                                                         |
| 207 | Multi-Status (WebDAV)         | Сообщение содержит несколько статусов для разных частей запроса                                               |
| 208 | Already Reported (WebDAV)     | Повторяющиеся зависимости уже были перечислены в предыдущем ответе.                                           |
| 226 | IM Used (HTTP Delta Encoding) | Сервер выполнил `GET`-запрос, и ответ представляет результат одного или нескольких манипуляций с экземпляром. |

3xx коды относятся к редиректам.     (3xx: Redirection)

| **Код** | **Статус**         | **Описание**                                                      |
| ------- | ------------------ | ----------------------------------------------------------------- |
| 301     | Moved Permanently  | Ресурс перемещен навсегда.                                        |
| 302     | Found              | Временное перенаправление.                                        |
| 303     | See Other          | Ответ на запрос находится по другому адресу (всегда ведёт к GET). |
| 304     | Not Modified       | Ресурс не изменился с последнего обращения.                       |
| 305     | Use Proxy          | Запрос к ресурсу должен осуществляться через прокси.              |
| 307     | Temporary Redirect | Временное перенаправление с сохранением метода.                   |
| 308     | Permanent Redirect | Постоянное перенаправление с сохранением метода.                  |
4xx коды указывают на ошибки со стороны клиента.
(4xx: Client Error)

| Код | Статус                               | Описание                                                                                             |
| --- | ------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| 400 | Bad Request                          | Некорректный запрос.                                                                                 |
| 401 | Unauthorized                         | Требуется аутентификация.                                                                            |
| 403 | Forbidden                            | Доступ запрещен.                                                                                     |
| 404 | Not Found                            | Ресурс не найден.                                                                                    |
| 405 | Method Not Allowed                   | Метод (GET, POST и т.д.) не поддерживается для данного URL (но поддерживается для других).           |
| 406 | Not Acceptable                       | Сервер не может выдать ответ в формате, указанном в заголовках `Accept` клиента.                     |
| 407 | Proxy Authentication Required        | Требуется аутентификация для прокси-сервера.                                                         |
| 408 | Request Timeout                      | Сервер не дождался полного запроса от клиента в течение установленного времени.                      |
| 409 | Conflict                             | Запрос конфликтует с текущим состоянием сервера.                                                     |
| 410 | Gone                                 | Ресурс удален навсегда.                                                                              |
| 411 | Length Required                      | Для запроса требуется указать заголовок `Content-Length`.                                            |
| 412 | Precondition Failed                  | Не выполнено одно из условий, заданных в заголовках запроса (например, `If-Match`).                  |
| 413 | Payload Too Large                    | (ранее "Request Entity Too Large") — Тело запроса превышает лимит, установленный сервером.           |
| 414 | URI Too Long                         | Длина запрашиваемого URL превышает лимит сервера.                                                    |
| 415 | Unsupported Media Type               | Сервер не поддерживает формат данных, указанный в теле запроса.                                      |
| 416 | Range Not Satisfiable                | Запрошенный диапазон (в `Range`) не может быть выполнен (выходит за пределы файла).                  |
| 417 | Expectation Failed                   | Сервер не может удовлетворить условию, указанному в заголовке `Expect`.                              |
| 418 | I'm a teapot (RFC 2324, шуточный)    | Сервер — чайник, не может заваривать кофе.                                                           |
| 421 | Misdirected Request                  | Запрос был отправлен на сервер, не способный дать ответ (проблема с конфигурацией).                  |
| 422 | Unprocessable Entity                 | Запрос синтаксически корректен, но содержит семантические ошибки (например, невалидное поле).        |
| 423 | Locked                               | Ресурс, к которому обращаются, заблокирован.                                                         |
| 424 | Failed Dependency                    | Запрос не выполнен из-за неудачи предыдущего запроса.                                                |
| 425 | Too Early                            | Сервер не хочет рисковать обрабатывать запрос, который может быть повторён (из-за атаки повторения). |
| 426 | Upgrade Required                     | Сервер требует перехода на другой протокол (например, TLS/1.3).                                      |
| 428 | Precondition Required                | Сервер требует, чтобы запрос был условным (например, с `If-Match`).                                  |
| 429 | Too Many Requests                    | Слишком много запросов.                                                                              |
| 431 | Request Header Fields Too Large      | Заголовки запроса слишком велики для обработки.                                                      |
| 450 | Blocked by Windows Parental Controls | Доступ заблокирован родительским контролем Windows.                                                  |
| 451 | Unavailable For Legal Reasons        | Доступ к ресурсу закрыт по юридическим причинам (цензура, авторские права).                          |
5xx коды свидетельствуют о проблемах на стороне сервера.      (5xx: Server Error)

| Код | Статус                          | Описание                                                                                                     |
| --- | ------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| 500 | Internal Server Error           | Внутренняя ошибка сервера.                                                                                   |
| 501 | Not Implemented                 | Сервер не поддерживает функциональность, необходимую для выполнения запроса (например, неизвестный метод).   |
| 502 | Bad Gateway                     | Ошибки шлюза.                                                                                                |
| 503 | Service Unavailable             | Сервис недоступен.                                                                                           |
| 504 | Gateway Timeout                 | Таймаут шлюза.                                                                                               |
| 505 | HTTP Version Not Supported      | Версия HTTP не поддерживается.                                                                               |
| 506 | Variant Also Negotiates         | На сервере произошла ошибка конфигурации: выбранный вариант ресурса сам инициирует согласование содержимого. |
| 507 | Insufficient Storage            | Не удалось выполнить запрос из-за нехватки места на сервере.                                                 |
| 508 | Loop Detected                   | Сервер обнаружил бесконечный цикл при обработке запроса.                                                     |
| 509 | Bandwidth Limit Exceeded        | Исчерпана пропускная способность                                                                             |
| 510 | Not Extended                    | Для доступа к ресурсу требуется расширить запрос.                                                            |
| 511 | Network Authentication Required | Требуется аутентификация в сети                                                                              |
| 526 | Invalid SSL Certificate         | Недействительный SSL-сертификат.                                                                             |
| 529 | Site is overloaded              | Сайт перегружен.                                                                                             |
| 530 | Site is frozen                  | Сайт "заморожен"                                                                                             |
| 598 | Network Read Timeout            | Неформальный код, используемый некоторыми прокси для таймаута чтения сети.                                   |

Протоколы (Protocols)


| protocol                  | name                                               | описание                                                                                                                                                                 |
| ------------------------- | -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [Ethernet](prot/Ethernet.md) | IEEE 802.3                                         | Основа большинства локальных проводных сетей (LAN). Определяет формат кадров и доступ к среде.                                                                           |
| [Wi-Fi](prot/Wi-Fi.md)       | IEEE 802.11                                        | Набор стандартов для беспроводных локальных сетей (WLAN).                                                                                                                |
| [PPP](prot/PPP.md)           | Point-to-Point Protocol                            | Протокол для установления прямого соединения между двумя узлами, часто используется dial-up и DSL-подключениях.                                                          |
| [ARP](prot/ARP.md)           | Address Resolution Protocol                        | Протокол для преобразования IP-адреса (сетевого уровня) в MAC-адрес (канального уровня) в локальной сети.                                                                |
| [ICMP](prot/ICMP.md)         | Internet Control Message Protocol                  | Протокол для передачи служебных и диагностических сообщений, например, для утилиты ping или сообщений об ошибках маршрутизации.                                          |
| [TCP](prot/TCP.md)           | Transmission Control Protocol                      | Надёжный протокол с установлением соединения.                                                                                                                            |
| [UDP](prot/UDP.md)           | User Datagram Protocol                             | Быстрый протокол без установления соединения (датаграммный). Не гарантирует доставку и порядок пакетов. Используется для потокового видео, онлайн-игр, DNS-запросов.     |
| [SCTP](prot/SCTP.md)         | Stream Control Transmission Protocol               | Протокол, сочетающий черты TCP и UDP. Обеспечивает надёжную доставку с поддержкой мультиплексирования нескольких потоков. Часто используется в телекоммуникациях.        |
| [HTTP](prot/HTTP.md)         | HyperText Transfer Protocol                        | Фундаментальный протокол для передачи данных во Всемирной паутине (WWW). Используется для загрузки веб-страниц и работы с API.                                           |
| [HTTPS](prot/HTTPS.md)       | Hypertext Transfer Protocol Secure                 | Безопасная версия HTTP, где данные шифруются с помощью протоколов TLS/SSL.                                                                                               |
| [FTP](prot/FTP.md)           | File Transfer Protocol                             | Протокол для передачи файлов между клиентом и сервером. Имеет раздельные каналы для данных и команд. Считается устаревшим и небезопасным.                                |
| [SSH](prot/SSH.md)           | Secure Shell                                       | Криптографический сетевой протокол для безопасного удалённого управления операционной системой, туннелирования и передачи файлов (SFTP, SCP).                            |
| [Telnet](prot/Telnet.md)     |                                                    | Протокол для удалённого доступа к командной строке сервера. Передаёт все данные (включая пароли) в открытом виде, поэтому небезопасен.                                   |
| [RDP](prot/RDP.md)           | Remote Desktop Protocol                            | Проприетарный протокол от Microsoft для подключения к удалённому рабочему столу с графическим интерфейсом.                                                               |
| [SMTP](prot/SMTP.md)         | Simple Mail Transfer Protocol                      | Протокол для отправки исходящей почты от клиента к серверу и для передачи почты между серверами.                                                                         |
| [POP3](prot/POP3.md)         | Post Office Protocol v3                            | Протокол для загрузки почты с сервера на локальное устройство. Обычно после загрузки письма удаляются с сервера.                                                         |
| [IMAP](prot/IMAP.md)         | Internet Message Access Protocol                   | Протокол для управления почтой непосредственно на сервере. Позволяет синхронизировать состояние писем (прочтение, папки) между несколькими устройствами.                 |
| [DNS](./prot/DNS.md)           | Domain Name System                                 | Распределённая система, преобразующая удобные для человека доменные имена (например, [google.com](https://google.com)) в машинные IP-адреса.                             |
| [DHCP](prot/DHCP.md)         | Dynamic Host Configuration Protocol                | Протокол для автоматического назначения IP-адресов и других сетевых параметров (маска, шлюз, DNS) устройствам при подключении к сети.                                    |
| [LDAP](prot/LDAP.md)         | Lightweight Directory Access Protocol              | Протокол для доступа и управления службами каталогов, которые хранят информацию о пользователях и ресурсах в сети.                                                       |
| [SNMP](prot/SNMP.md)         | Simple Network Management Protocol                 | Протокол для управления и мониторинга сетевых устройств (маршрутизаторы, коммутаторы, серверы).                                                                          |
| [SIP](prot/SIP.md)           | Session Initiation Protocol                        | Протокол сигнализации для установления, изменения и завершения сеансов связи, таких как VoIP-звонки и видеоконференции.                                                  |
| [RTP](prot/RTP.md)           | Real-time Transport Protocol                       | Протокол для фактической передачи аудио- и видеопотоков в реальном времени. Работает поверх UDP.                                                                         |
| [RTSP](prot/RTSP.md)         | Real Time Streaming Protocol                       | Протокол управления для потоковой передачи медиаданных (пауза, воспроизведение, запись).                                                                                 |
| [NFS](prot/NFS.md)           | Network File System                                | Протокол для доступа к файлам по сети в UNIX- и Linux-системах, позволяющий монтировать удалённые каталоги.                                                              |
| [SMB/CIFS](prot/SMB-CIFS.md) | Server Message Block / Common Internet File System | Сетевой протокол для общего доступа к файлам, принтерам и другим ресурсам в сетях Windows.                                                                               |
| [WebSocket](prot/WebSocket.md) |                                                    | Протокол, обеспечивающий постоянное двунаправленное соединение (full-duplex) между клиентом и сервером поверх одного TCP-соединения. Используется для чатов, онлайн-игр. |
| [MQTT](prot/MQTT.md)        | Message Queuing Telemetry Transport                | Легковесный протокол обмена сообщениями по модели "издатель-подписчик" (publish-subscribe). Широко используется в Интернете вещей (IoT).                                 |
| [IP](prot/IP.md)             | Internet Protocol                                  | Основной протокол сетевого уровня стека TCP/IP. Отвечает за адресацию (IPv4, IPv6) и маршрутизацию пакетов между сетями.                                                 |
| [BGP](prot/BGP.md)           | Border Gateway Protocol                            | Протокол маршрутизации между автономными системами (разными провайдерами). Фактически "склеивает" интернет в единое целое.                                               |
| [OSPF](prot/OSPF.md)         | Open Shortest Path First                           | Протокол динамической маршрутизации внутри одной автономной системы. Использует алгоритм Дейкстры для поиска кратчайшего пути.                                           |
| [RIP](prot/RIP.md)           | Routing Information Protocol                       | Один из старейших протоколов дистанционно-векторной маршрутизации внутри автономной системы. Простой, но малоэффективен для больших сетей.                               |
| [TLS/SSL](prot/TLS-SSL.md)   | Transport Layer Security / Secure Sockets Layer    | Криптографические протоколы, обеспечивающие безопасную связь поверх TCP. TLS - современная версия, пришедшая на смену SSL.                                               |
| [IPsec](prot/IPsec.md)       | Internet Protocol Security                         | Набор протоколов для защиты IP-пакетов на сетевом уровне. Используется для создания VPN.<br>                                                                             |
| [WireGuard](prot/WireGuard.md) |                                                    | Современный, простой и высокопроизводительный протокол VPN. Работает на уровне ядра операционной системы.                                                                |
| [OpenVPN](prot/OpenVPN.md)   |                                                    | Популярный протокол VPN с открытым исходным кодом, гибкий и использующий TLS для шифрования.                                                                             |
| [WPA2/WPA3](prot/WPA2-WPA3.md) | Wi-Fi Protected Access                             | Протоколы безопасности для защиты беспроводных сетей Wi-Fi. Шифруют трафик между устройством и точкой доступа.                                                           |
| [NTP](prot/NTP.md)           | Network Time Protocol                              | Протокол для синхронизации системного времени компьютеров в сети с высокой точностью.                                                                                    |
| [QUIC](prot/QUIC.md)         | Quick UDP Internet Connections                     | Современный транспортный протокол от Google, работающий поверх UDP. Объединяет установление соединения и шифрование, уменьшая задержки. Является основой для HTTP/3.     |
| [gRPC](prot/gRPC.md)         | gRPC Remote Procedure Calls                        | Современный фреймворк для вызова удалённых процедур (RPC), использующий HTTP/2 в качестве транспорта и Protocol Buffers для описания интерфейсов и сериализации данных.  |
